# RubiKey 当前状态速记

本文基于 `2026-03-26` 仓库内实际代码整理，定位以“趣味工具”发布为主，只保留对体验影响较大的信息。

## 1. 现在能做什么

- 已支持 GAN `v2 / v3 / v4`
- 已支持 Moyu32 / 魔域新协议
- 已打通主链路：

`智能魔方 -> 蓝牙连接 -> 协议解析 -> move 归一化 -> Profile 查找 -> 主进程宏执行 -> Windows 输入`

- 支持 12 个基础转动映射
- 支持多 Profile
- 支持键盘 / 鼠标动作
- 支持多步宏
- 支持紧急停止
- 支持托盘与全局快捷键
- 默认顶部菜单栏已隐藏
- 支持基础陀螺仪鼠标
- 支持合并动作日志（未绑定显示转动，已绑定显示执行详情）
- 方案映射页已调整为“先选转动、再编辑当前规则”的更直观布局

补充：

- Moyu32 基础真机验证已完成，连接、move 与 gyro 数据链路当前未发现阻塞性问题
- 这个项目应优先参考手动真机测试结果，不需要过度强调工程形式上的完备度

## 2. 目前最影响体验的问题

- 被动断开后，UI 还没有完全同步回“已断开”状态
- MAC 自动获取仍可能失败，部分设备还需要手动输入
- 连接失败、断开恢复、重连提示还不够完整
- 陀螺仪 `game` 模式在 Minecraft 真机里暴露出姿态轴映射问题，现已暂时从前端隐藏

## 3. 发布视角下的完成度判断

如果按“趣味工具”来看，当前状态可以概括为：

- 主链路已经可用
- 多品牌接入已经落地
- 基础映射和宏执行已经够玩
- 但连接一致性、陀螺仪体验和发布说明还需要补一层

保守看，大致可以认为：

- 核心可玩度：`80%+`
- 发布友好度：`60% - 70%`

## 4. 现在最值得继续补的事

1. 连接状态一致性
   - 被动断开后的 UI 回写
   - 连接失败和恢复提示
2. 发布说明
   - 已验证设备清单
   - MAC 获取说明
   - 陀螺仪推荐参数
   - 已知问题与回避方式
3. 陀螺仪 `game` 模式
   - 修正姿态映射模型
   - 修好后再重新开放

## 5. 一句话结论

RubiKey 已经不只是 Demo，而是一个可以发布给喜欢折腾智能魔方映射玩法的趣味工具；当前最该补的不是工程排面，而是连接体验、说明文档和陀螺仪细节。
