# Moyu32 现状速记

本文基于 `2026-03-26` 仓库内实际代码整理，只保留和体验最相关的结论。

## 1. 当前结论

Moyu32 这一块已经接入完成并能正常使用：

- 已进入统一接入层
- 已接入现有 move -> Profile -> 宏执行链路
- 已具备 V10 AI 风格陀螺仪鼠标支持

补充：

- 基础连接、move、宏触发与 gyro 数据链路的真机验证已完成
- 这一块优先看手动真机测试结果，不追求全设备覆盖或自动化完整度

## 2. 已经能用的部分

- Moyu32 转动可进入动作日志，已绑定动作时会直接显示执行详情
- 对应 move 可以触发 Profile 匹配与宏执行
- 连接诊断可看到品牌、协议、设备名、MAC
- 对支持陀螺仪的设备，可以请求启用 gyro
- gyro 数据会同时送入渲染层预览与主进程鼠标控制器

## 3. 当前最影响体验的问题

- 被动断开时，UI 状态还没有完全统一回写
- MAC 自动获取仍可能失败，部分场景需要手动输入
- `game` 模式在 Minecraft 真机里暴露出姿态轴映射问题，当前表现不符合正常第一人称视角控制
- 为避免误导使用，前端已暂时隐藏 `game` 模式入口，并将配置回退为 `desktop`

## 4. 简单完成度判断

如果按“趣味工具里的一个已可用设备分支”来看：

- 接入完成度：`90%+`
- 真机可用度：`85%+`
- 发布友好度：`60% - 70%`

## 5. 后续最值得补的事

1. 被动断开后的状态同步
2. 修正陀螺仪 `game` 模式的姿态映射，再重新开放入口
3. 整理已验证型号、推荐参数和已知问题
